# LLM Foundational

- **Definición**: Se refiere a un LLM general que está listo para su uso en tareas diversas, sin requerir ajustes adicionales. Estos modelos son generalmente grandes, versátiles y pueden abordar una amplia gama de problemas con instrucciones adecuadas (*prompt engineering*).
- **Ventajas**:
    - Flexibilidad para múltiples tareas.
    - Puede generalizar mejor cuando se enfrenta a datos fuera del dominio del entrenamiento.
- **Ejemplo**: GPT-4, que puede responder tanto preguntas generales como generar código o analizar texto legal con el contexto adecuado.

### **Diferencias Clave**

| Aspecto | LLM | LLM Funcional |
| --- | --- | --- |
| ***Estado*** | General, no optimizado. | Adaptado o ajustado para tareas prácticas. |
| ***Uso directo*** | Requiere ajustes o ingeniería de *prompts*. | Listo para ser usado en aplicaciones específicas. |
| ***Adaptación*** | Puede necesitar *fine-tuning* para ciertas tareas. | Puede ser un modelo preajustado o diseñado con configuraciones listas. |
| ***Ejemplo de tarea*** | Generación de texto genérica. | Asistente para un dominio específico, como soporte técnico o análisis legal. |
